Pic-to-Bin Open the app →

Phone photos → 3D-printed Gridfinity bin.

Photograph your tools on a printed template. The app traces each outline, packs them into the smallest Gridfinity bin that fits, and hands you a parametric Fusion 360 model ready to slice.

You'll need

  • Autodesk Fusion 360

    Free for personal use. Download from Autodesk →

  • A 3D printer

    Any FDM printer that handles standard Gridfinity bins (PLA, PETG, or ABS).

  • Paper + a regular printer

    Print the ArUco-marker template once on plain paper.

  • A phone camera

    Any modern phone. Moderate angles are fine — perspective gets corrected automatically.

Step 0 · One-time setup

Two short things to do once. After this, every bin starts at Step 1.

① Install the Fusion 360 add-in

Download the add-in and drop one folder into your Fusion API directory. INSTALL.txt inside the ZIP has the exact paths for Windows and macOS — about 30 seconds.

⬇ Download Fusion add-in (.zip)

Show install steps
  1. Extract pic-to-bin-fusion.zip. Inside you'll see an AddIns/pic_to_bin/ folder.
  2. Open your Fusion 360 API directory:
    • Windows: %APPDATA%\Autodesk\Autodesk Fusion 360\API
    • macOS: ~/Library/Application Support/Autodesk/Autodesk Fusion 360/API
  3. Drag the ZIP's AddIns/pic_to_bin folder into the API's AddIns/ folder (so the path ends with …/AddIns/pic_to_bin/pic_to_bin.py).
  4. In Fusion 360, press Shift+S, switch to the Add-Ins tab, select pic_to_bin, and click Run. Tick "Run on Startup" so the button shows up every session.
  5. In a Design workspace, the new button appears under Solid → Create → Gridfinity Pic-to-Bin.

② Print the ArUco template

Print on plain paper at 100 % scale (no fit-to-page). One template is reusable — print once and photograph as many tools as you like. Pick the paper size you'll load; the green chroma-key variant helps SAM2 segment light or reflective tools.

How it works

A typical run takes 5–10 minutes end to end.

  1. 1

    Photograph your tools

    Lay each tool on the template you printed in Step 0 and snap a phone photo. One photo per tool. Overhead light, all 8 markers in frame when possible.

  2. 2

    Enter each tool's height

    Measure thickness with calipers or a ruler — that sets the pocket depth. The bin auto-sizes to the tallest tool.

  3. 3

    Inspect the trace

    The app overlays its trace on each photo. Adjust trace parameters or rerun a specific image if a detail looks off.

  4. 4

    Review the auto-packed layout

    Tools snap to the smallest Gridfinity bin that fits. Print the 1:1-scale PDF fit-test on paper and check the physical tools sit cleanly before committing to a 3D print.

  5. 5

    Download bin_config.json

    The bin recipe. Polygons, dimensions, and Gridfinity sizing the Fusion add-in reads in one click.

  6. 6

    Build the model in Fusion 360

    Click Solid → Create → Gridfinity Pic-to-Bin, pick the JSON. Fusion builds the parametric body, stacking lip, tool pockets, finger slots, and Gridfinity base — all in named timeline groups you can edit afterward.

  7. 7

    Export as STL (or mesh)

    Fusion's File → 3D Print writes an STL ready for any slicer. STEP works too if you want to remix the geometry.

  8. 8

    3D-print and drop in your tools

    Slice and print at your usual settings. ~2 mm of clearance is baked into each pocket so tools drop in cleanly without forcing.
